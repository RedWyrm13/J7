# J7 Sampling Project

Classification of quantum circuit families using statistical distribution fingerprinting, with application to electromagnetic spectrum monitoring.

## Overview

Three quantum circuit families are studied:
- **IQP** (Instantaneous Quantum Polynomial): diagonal gates sandwiched by Hadamard layers; classically hard to sample from.
- **Clifford**: generated by {H, S, CNOT}; efficiently simulable classically via the stabilizer formalism.
- **Clifford+T**: universal gate set; breaks stabilizer structure and approaches quantum advantage with sufficient T-count.

Each family produces a characteristic probability distribution over measurement outcomes. We extract a fixed set of statistical features from these distributions and train classical classifiers to distinguish the families. The same pipeline is then applied to a contrived spectrum dataset, testing whether the learned representations transfer to a spectrum monitoring setting.

## Feature Set

Features are extracted from the raw measurement counts dictionary for each circuit run. Only features whose estimation cost scales **polynomially** with qubit count are retained.

| Feature | Size | Shot cost | Notes |
|---|---|---|---|
| Hamming-weight histogram | n+1 | O(n) | P(weight = w) for w = 0..n |
| Single-qubit marginals | n | O(n) | P(qubit i = 1) for each qubit |
| Parity bias | 1 | O(1) | P(even parity) − P(odd parity) |
| Nearest-neighbor ZZ correlations | n−1 | O(n²) | ⟨ZᵢZᵢ₊₁⟩ − ⟨Zᵢ⟩⟨Zᵢ₊₁⟩ |

**Total feature vector size: 3n + 1** (linear in qubit count).

### Why entropy and collision probability are excluded

Shannon entropy requires O(2ⁿ/n) shots for accurate estimation because the empirical estimator is biased whenever outcomes go unobserved—which is the norm above ~10 qubits. Collision probability requires O(2^(n/2)) shots (the birthday problem threshold). Both grow exponentially with qubit count, which would negate the resource efficiency motivation of the approach. Biased estimates of these quantities do retain some discriminative signal, but at the cost of introducing a systematic shot-count dependence that cannot be controlled at fixed quadratic budget.

## Shot Scaling

The shot budget is set to:

```
shots = shots_base × n_qubits²
```

With `shots_base = 16` (default):

| Qubits | Shots |
|--------|-------|
| 4 | 256 |
| 8 | 1024 |
| 10 | 1600 |
| 15 | 3600 |
| 20 | 6400 |

This matches the O(n²) cost of the ZZ correlation suite, providing consistent statistical quality for all retained features across the full range.

## Configuration

Both pipeline stages are driven by YAML files with a shared format:

```yaml
qubit_range: [4, 21]       # range(4, 21) → qubits 4..20 inclusive
labels: ['iqp', 'clifford', 'clifford_t']
shots_base: 16             # shots = shots_base * n_qubits^2
n_ops: 1000
n_circuits: 1000
resamples_per_circuit: 1
master_seed: 42
snapshots_per_session: 50  # spectrum only
```

## Running the Pipeline

### Step 1: Generate circuit data

```bash
cd main/
python3 generate_distributions.py circuit_configuration.yaml
```

On HPC via SLURM:
```bash
cd main/
sbatch generate_distributions.slurm
```

Output goes to `data/quantum/` as `.npz` files named:
```
circuitFamily_{family}_qubits{n}_ops{ops}_shotsPerDatapoint{s}_numCircuits{N}_resamplesPerCircuit{r}_masterSeed{seed}.npz
```

### Step 2: Generate contrived spectrum data

Requires Step 1 to be complete for the relevant qubit range.

```bash
cd spectrum/
python3 generate_contrived_spectrum.py spectrum_configuration.yaml
```

Output goes to `data/spectrum/`.

### Step 3: Run classification experiments

```bash
cd spectrum/
python3 run_spectrum_experiment.py spectrum_configuration.yaml
```

This runs three experiments for each qubit count:
1. **Circuit-only** — train and test on raw circuit data (baseline)
2. **Spectrum-only** — train and test on spectrum session data
3. **Cross** — train on circuit data, test on spectrum session data (key result)

Plots are saved to `spectrum/plots/spectrum_experiment_qubits{n}.png`.

## Classifiers

Four scikit-learn classifiers are evaluated:
- **Logistic Regression** (L-BFGS, max_iter=1000)
- **Decision Tree** (unlimited depth)
- **Gradient Boosting**
- **Extra Trees** (200 estimators)

Random-chance baseline for 3 classes: 0.333.

## Key Result: Cross-Domain Generalization

The cross experiment reveals a consistent pattern: **tree-based classifiers dominate in-distribution but fail to generalize to the spectrum domain; Logistic Regression is the only classifier that reliably exceeds random chance on cross data**.

This is explained by the averaging construction used to build spectrum sessions (50 circuit feature vectors are averaged per session). Averaging preserves class means but collapses feature variance. Tree methods rely on absolute threshold values calibrated to the training variance, which break under this shift. Logistic Regression learns a separating hyperplane direction that remains valid because class means are unchanged under averaging.

This result implies that the linear structure of the feature space—the direction separating the three circuit families—survives the spectrum construction, and is therefore the relevant signal for spectrum monitoring applications.
